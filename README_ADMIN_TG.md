# Что добавлено

1. После 3–5 завершённых игр игрок блокируется и видит окно перехода на партнёрский сайт.
2. Если демо-счёт 10$ закончился раньше 5 игр, игрок тоже блокируется и видит это окно.
3. После блокировки доска и кнопки игры отключаются. Продолжить можно только через кнопку партнёрского сайта.
4. Если игрок нажал кнопку перехода и вернулся в Telegram WebApp, окно снова показывает текст: «Игру можно продолжить на сайте».
5. Добавлен Cloudflare Worker для статистики, Telegram webhook и админки по `/admin`.
6. Данные игроков теперь хранятся в Cloudflare D1 Database, а не в KV.

# Важно

Статистика, дожим и сброс игроков невозможны только на статическом GitHub + Cloudflare Pages без серверной части. Поэтому добавлен `worker.js` — это Cloudflare Worker, он тоже работает через Cloudflare и не требует VPS/хостинга.

# Настройка Cloudflare D1

1. Загрузите обновлённый проект в GitHub.
2. Frontend можно оставить как Cloudflare Pages.
3. Создайте базу данных:
   - Cloudflare Dashboard → Workers & Pages → D1 SQL Database → Create database.
   - Название можно поставить `mines_db`.
4. Создайте Cloudflare Worker из файла `worker.js`.
5. В настройках Worker откройте Settings → Bindings.
6. Добавьте D1 Database binding:
   - Variable name: `DB`
   - D1 database: ваша база `mines_db`
7. В настройках Worker добавьте переменные:
   - `PARTNER_URL` — ваша партнёрская ссылка.
   - `TELEGRAM_BOT_TOKEN` — токен бота от BotFather.
   - `ADMIN_IDS` — Telegram ID администраторов через запятую, например `123456789,987654321`.
   - `SETUP_SECRET` — любой длинный секретный текст.
8. Подключите Worker route к тому же домену, где игра, минимум для путей:
   - `/api/*`
   - `/bot`
   - `/set-webhook`

# Нужно ли вручную создавать таблицу?

Необязательно. Worker сам создаёт таблицу `players` при первом запросе.

Если хотите создать таблицу вручную, выполните SQL из файла `d1_schema.sql` в D1 Console.

# Установка webhook бота

Откройте в браузере:

`https://ВАШ-ДОМЕН/set-webhook?secret=ВАШ_SETUP_SECRET`

После успешного ответа Telegram будет отправлять команды бота в Worker.

# Админка в Telegram

Отправьте боту `/admin` с аккаунта, ID которого указан в `ADMIN_IDS`.

В админке есть:
- всего игроков;
- игроков за 24 часа;
- сколько перешли по ссылке;
- кнопка «Обновить»;
- кнопка «Дожим» — отправляет сообщение тем, у кого окно уже появилось, но они не нажали партнёрскую ссылку;
- кнопка «Сброс игрока» — бот подскажет команду `/reset USER_ID`.

# Как узнать USER_ID игрока

В Telegram WebApp используется `tg.initDataUnsafe.user.id`. Для теста вне Telegram используется `demo-user`.
